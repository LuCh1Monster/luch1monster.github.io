    XShell连接Docker服务器下Centos终端

    工作中使用的服务器是使用 portainer框架搭建的 docker 服务器 Debian 容器，因为主要是使用 Jupyter Notebook 来工作，且 portainer 已经做了端口映射。但是发现 由于镜像的原因，像一些基本的 Linux 命令都无法正常的使用，例如 free -m 等，且只有 web 一个 terminal 终端，不支持多开，这对于一个用惯了常规物理服务器的 coder，感觉非常不舒服。

    这里将总的 Docker 服务器称为 server-master，我自己申请的 Linux 服务器称为 server-me。假定 server-master 的 IP 为 1.1.1.1，server-me 的 IP 为 2.2.2.2。 这里还有一个非常大的问题就是，我本地的机器(local)在内网的 IP 为 3.3.3.3，机器之间是不能完全 ping 通的。具体如下所示:

    local         ->      server-master
local         -x-     server-me
server-me     -x-     local
server-me     ->        server-tmp
server-tmp     -x-        server_me

    具体关系如上所示，关于 server-master 的终端是无法登陆和知晓的，我们只知道 server-master 的 IP ，且没有权限登陆 server-master 的终端的。

    摸索了很久，终于完成了使用本地Windows电脑连接 server-me 服务器终端，这里分享一下。

    1. 制作镜像

    前面提到之前使用的 docker 容器一些常规的 bash 都没有，这里需要先制作镜像，这里选用的是网易 163 的镜像，地址为 : https://c.163.com/hub#/m/home/ ，选择 Centos 6.7 作为我们的 docker 镜像。


    点击 Build the Image 就可以制作镜像了。

    2. 搭建 docker 服务器 Centos6.7

    点击 “Add container”，填写以下内容。注意一定要端口多映射出来，这里是将 server-master 的端口映射到 server-me 的端口上，比如 1.1.1.1:58888 映射到 2.2.2.2:8888 端口上，最终我们访问的地址其实也只能通过 http://1.1.1.1:58888 进行访问，所以多映射出端口出来以便后续使用。注意: 22、80、3306、8000 和 8888 端口一定要映射出来，22端口是 Linux 服务器默认的 ssh 连接端口，这是后面使用 XShell 连接 Centos6.7 必要的，不然会多一步修改连接端口的步骤。


    下面进行数据盘的挂载，填写以下内容，可以在 Network 中填写 server-me 的 hostname，点击 “Deploy the container” 等待一会服务器就创建好了，创建好后服务器会自动启动。


    打开容器列表，可以发现你的服务器已经搭建成功了。

    3. 安装服务

    由于镜像中还是存在一些常规的命令无法使用，比如 passwd 等，需要手动安装一些服务。执行以下代码，进行服务的安装:

    → yum update
→ sudo yum install -y openssh-clients openssh-server
→ sudo yum install -y passwd scp bzip2
→ sudo yum install -y vixie-cron crontabs
→ sudo yum install -y vim wget gcc g++ git git-core make cmake

    4. 添加root密码和用户

    • 先给 root 设置密码
    → passwd root
    • 添加用户，并指定目录
    → useradd -d /opt/foo -m foo
→ passwd foo
    • 修改 foo 用户的权限，这时候发现 /etc/sudoers 这个文件不存在，使用 sudo 命令会提示 bash: sudo: command not found，这就导致无法修改 foo 用户的权限。需要先安装 sudo 服务，代码如下:
    → yum install sudo
    • 修改 foo 用户权限为 root 权限，在 root 下面添加一行
    → vim /etc/sudoers

# 下面为修改的内容
root ALL=(ALL)    ALL
foo  ALL=(ALL)    ALL

    5. 使用 XShell 连接 Centos

    首先使用 xshell 生成本机的密钥，你也可以使用 Git-Gui 来生成。






    最后保存成文件如：id_rsa_2048.pub，然后填写生成公钥过程中输入的密码

    上传公钥文件到 linux 服务器的 ~/.ssh/ 目录，执行命令 cat id_rsa_2048.pub >> authorized_keys，如果没有 authorized_keys 文件，则自己创建一个。修改权限 chmod 600 authorized_keys。

    → cat id_rsa_2048.pub >> authorized_keys
→ chmod 600 authorized_keys


    点击确定后，就可以连接使用了。对于 XShell 连接 foo 用户，方法都是一样的，不再赘述。
