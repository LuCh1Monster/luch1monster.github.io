    1. 使用Hexo、GitHub建站

    关于如何使用 Hexo 和 GitHub 搭建个人站点，如何在自己的个人小站上使用 3-hexo 主题，在原作者的网站上有详细的介绍，大家可以去访问作者个人网站 https://yelog.org/ ，这里感谢作者将其开源出来，非常优秀的作品。

    这里主要分享一下如何修改哪些文件内容来完全构建属于自己的小站，以及为了页面美化所做的小部分源码修改。

    2. 需要安装的插件

    作者做了很多有用的插件功能，需要安装对应的插件，修改对应的文件内容才可以开启这些功能，这里大家可以先安装上，安装需要在项目的根目录下执行命令。

    • 置顶插件
    → npm install hexo-generator-topindex --save
    • WordCount 插件: 需要在项目根文件下执行命令
    → npm i hexo-wordcount --save
    • RSS 订阅
    → npm install hexo-generator-feed --save
    • Mermaid 图标
    → npm install hexo-filter-mermaid-diagrams
    • 全文搜索
    → npm install hexo-generator-search --save
    • 文件排序
    → npm install hexo-generator-index-pin-top --save

    3. 文件修改

    3.1 hexo项目配置文件

    这里我初始化的项目名为 blog，首先修改 blog/_config.yml 文件，修改文件内容如下:

    # Site
title: 路痴大魔王
subtitle: 'Hello, coder!'
description: ''
keywords:
author: 郁明敏
language: zh-CN
timezone: 'Asia/Shanghai'

# URL
url: https://www.monsteryu.top
...


# Writing
...
highlight:
  enable: false
  # line_number: false
  # auto_detect: false
  # tab_replace: ''

# Extensions
theme: 3-hexo

# Search All
search:
  path: search.xml
  field: post

    这里对几点修改做一下解释:

    • Site 部分修改的网站基础信息，替换成你自己的信息，就成你自己的个人小站了。
    • URL 部分修改主要是作者在文章上设置了原始链接，不修改这一部分的话，原始链接会显示为 http://yoursite.com/2019/11/27/3-hexo-usage/ ，显然这不是我们想要的。

    • Writing 部分主要是关闭 hexo 自带的高亮，使用主题的高亮功能，修改成上述内容即可。
    • Extensions 部分主要修改 hexo 使用的默认主题，默认为 landscape，这里修改为 3-hexo。
    • Search All 部分为手动添加的代码，作者设计了全文搜索的功能，需要在 blog/_config.yml 中手动添加这部分代码。

    3.2 项目添加 404 页面和 about 页面

    1. 新增 404 页面，执行下面命令，至于想要显示什么样 404 页面，大家可以自己根据自己实际需求修改。
    → hexo new page 404

    我的 404 页面内容如下，文件 blog/source/404/index.md

    ---
title: 404
date: {{ date }}
---

---
## 页面未找到！

    注意: 这里 404 页面目前只在访问 https://www.monsteryu.top/404 起作用，随意访问一个不存在的地址会显示

    1. 新增 about 页面，用于显示个人的基本信息。

​```

    应该这样使用:

    ​```python

    3.8 表格添加出现水平滚动条

    由于原主题的表格是没有水平滚动条的，如果表格超出了页面，会在页面最下方的出现水平的滚动条，这样每次只能用键盘控制页面来查看超出页面边界的表格内容。为了更方便的阅读博客，对主题的源代码进行了一点更改:

    修改 blog/themes/3-hexo/source/css/_partial/post.styl 文件，找到以下内容:

    table
  padding 0
  border-spacing 0
  border-collapse collapse       
  tr
    border-top 1px solid #cccccc
    ...

    修改为以下内容:

    table
  padding 0
  border-spacing 0
  border-collapse collapse
  display block
  width: 100%
  overflow-y hidden
  overflow-x auto          
  tr
    border-top 1px solid #cccccc
    ...

    td
       border 1px solid #ddd
       text-align left
       margin 0
       padding 6px 13px
       min-width 100px  /* 添加最小单元格宽度 */

    4. 数学公式换行显示

    由于hexo的MarkDown渲染器与MathJax有冲突，所以在使用之前需要修改两个地方。在你的项目下编辑 node_modules\marked\lib\marked.js 脚本，搜索 esacpe: 来定义到相应位置:

    var inline = {
  escape: /^\\([!"#$%&'()*+,\-./:;<=>?@\[\]\\^_`{|}~]),
  ...
};
// 修改为
var inline = {
  escape: /^\\([!"#$%&'()*+,\-./:;<=>?@\[\]_`~])/,
  ...
}

    上述操作还修改了公式中 { 和 } 的使用，不然你需要使用 \\{ 和 \\} 才会让公式真正显示。

    同时，也去掉了对 | 转义，不然你在使用条件概率 $P(y|x, t)$ 这样的公式会报错。

    使用方法:

    Q1 =
\\ 1 + 1
\\ =2

    $$
    Q1 =
    \\ 1 + 1
    \\ =2
    $$

    注意 : 编辑公式时尽量不要用 *，使用 \times 进行代替。

    5. 添加访问密码

    5.1 低级做法

    在你的 D:\blog\themes\3-hexo\layout\_partial\head.ejs 文件末尾添加如下代码:

    <script>
    (function(){
        if('{{ page.password }}'){
            if (prompt('请输入文章密码') !== '{{ page.password }}'){
                alert('密码错误！');
                history.back();
            }
        }
    })();
</script>

    其中 '' 也可以换成换成固定密码，比如 123456，也可以为每篇文章设置一个密码。当需要在每篇文章中设置一个密码时，需要这样设置:

    ---
title: Hexo文章简单加密访问
date: 2016-12-01 10:45:29
tags: hexo
categories: 博客
keywords:
    - Hexo
    - 加密
description: 文章访问密码：password
password: 123456
---

    5.2 使用hexo-blog-encrypt插件

    在项目根目录下，运行下面命令:

    → npm install hexo-blog-encrypt --save

    然后在项目根目录下的 _config.yml 文件末尾添加下面内容:

    # 加密文章
encrypt:
    enable: true

    如果你输入的密码没有反应，可能是 Chrome 采用了 https 过滤的方法，看看 Firefox 下访问是否成功，本人在 Firefox 下面是可以访问 的，但是 Chrome、Opera、Edge 都不行。

    如果你是要在本地禁用 Chrome 的 https 协议的话，可以在浏览器中访问 chrome://net-internals/#hsts ，在 Delete domain security policies 中添加你本地的 IP 就可以了。

    6.7 代码块复制功能

    首先下载第三方库 clipboard.js ，下载地址:

    • clipboard.js
    • clipboard.min.js 推荐

    在 blog\themes\3-hexo\source\js 下创建 clipboard.js 文件和 clipboard.min.js 文件，输入两个 js 文件内容。

    然后在 blog\themes\3-hexo\source\js 下创建 clipboard-use.js 文件，输入以下内容:

    /*页面载入完成后，创建复制按钮*/
!function (e, t, a) { 
  /* code */
  var initCopyCode = function(){
    var copyHtml = '';
    copyHtml += '<button class="btn-copy" data-clipboard-snippet="">';
    //fa fa-globe可以去字体库替换自己想要的图标
    copyHtml += '  <i class="fa fa-clipboard"></i><span>copy</span>';
    copyHtml += '</button>';

    $(".pjax pre code").wrapInner('<div class="code-block"></div>');
    $(".pjax pre code .code-block").before(copyHtml);

    new ClipboardJS('.btn-copy', {
        target: function(trigger) {
            return trigger.nextElementSibling;
        }
    });
  }
  initCopyCode();
}(window, document);

    在 blog\themes\3-hexo\layout\post.ejs 文件中，添加以下内容，注意需要添加在 </body> 之前:

    <script src="<%=theme.blog_path?theme.blog_path.lastIndexOf("/") === theme.blog_path.length-1?theme.blog_path.slice(0, theme.blog_path.length-1):theme.blog_path:'' %>/js/clipboard.min.js?v=<%=theme.version%>" ></script>
<script src="<%=theme.blog_path?theme.blog_path.lastIndexOf("/") === theme.blog_path.length-1?theme.blog_path.slice(0, theme.blog_path.length-1):theme.blog_path:'' %>/js/clipboard-use.js?v=<%=theme.version%>" ></script>

    在 blog\themes\3-hexo\source\css\style.styl 中添加以下内容:

    .hljs

    但是发现，只有在首次加载才会出现复制按钮，切换到新的文章后，复制按钮就消失了。

    修改了 blog\themes\3-hexo\source\js\clipboard-use.js 文件，内容如下:

    /*页面载入完成后，创建复制按钮*/
var initCopyCode = function(){
  var copyHtml = '';
  copyHtml += '<button class="btn-copy" data-clipboard-snippet="">';
  //fa fa-globe可以去字体库替换自己想要的图标
  copyHtml += '  <i class="fa fa-clipboard"></i><span>copy</span>';
  copyHtml += '</button>';

  $(".pjax pre code").wrapInner('<div class="code-block"></div>');
  $(".pjax pre code .code-block").before(copyHtml);

  new ClipboardJS('.btn-copy', {
      target: function(trigger) {
          return trigger.nextElementSibling;
      }
  });
}


!function (e, t, a) { 
  /* code */
  initCopyCode();
}(window, document);


function refreshBtnCopy () {
  setTimeout(function() {
    initCopyCode()
  }, 2000)
}

    然后在文章切换时，添加一个点击事件，用来触发 refreshBtnCopy 函数，修改 blog\themes\3-hexo\layout\_partial\nav-right.ejs:

    <a <% if(post.top){%>id="top"<%}%>

    修改成:

    <a onclick="refreshBtnCopy()" <% if(post.top){%>id="top"<%}%>

    这样就可以在切换文章时重新加载代码复制按钮。

    7. 运行

    在项目根目录下，执行以下代码:

    → hexo clean && hexo g && hexo s

    当然，你在 blog/_config.yml 如果配置了你的 GitHub 仓库的话，则可以使用以下命令，将代码推送到你的 GitHub 仓库中，这样就能让别人访问你的个人小站了。

    → hexo clean && hexo g && hexo d
