    1. 设定临时环境变量:

    → export SLUGIFY_USES_TEXT_UNIDECODE=yes
→ export AIRFLOW_HOME=~/bigdata/airflow
→ pip install apache-airflow
→ pip install apache-airflow[password]
→ pip install flask_bcrypt

    因为使用环境是 Python3.6.4，安装中出现 ERROR: flask 1.0.3 has requirement Jinja2>=2.10, but you'll have jinja2 2.8.1 which is incompatible. 报错信息，原因是版本不兼容:

    → pip install Jinja2==2.10

    1) AirFlow MySQL 安装

    → pip install apache-airflow[mysql]

    2. 创建 airflow 数据库

    > create database airflow;
> create user 'airflow'@'%' identified by '123123';
> grant all privileges on airflow.* to 'airflow'@'%'  identified by '123123';
> flush privileges;

    3. 初始化airflow

    → airflow initdb

    5. 启动 airflow

    执行以下命令，如果终端不报错，能访问 http://ip:9000 能看到界面就表示安装成功了。

    → airflow webserver -p 9000

    6. 启动和停止脚本

    最后编写两个 bash 脚本，start-airflow-all.sh 和 stop-airflow-all.sh 用于启动和停止 airflow 任务。

    • 启动脚本 : start-airflow-all.sh
    #!/bin/bash

nohup airflow webserver >> airflow.log &

nohup airflow scheduler >> airflow.log &

nohup airflow worker >> airflow.log &
    • 停止脚本 : stop-airflow-all.sh
    #!/bin/bash

nohup ps -ef|grep -Ei '(airflow-webserver)'| grep master | awk '{print $2}'|xargs -i kill {} > log1.txt &

nohup ps -ef|grep -Ei '(airflow-scheduler)'| grep master | awk '{print $2}'|xargs -i kill {} > log1.txt &

    7. 添加 airflow 用户

    编写一个 add_user.py 文件，用来添加 AirFlow 用户:

    import airflow, sys
from airflow import models, settings

    8. 设置右上角不显示UTC时间格式

    8.1 修改时区

    airflow 默认使用 UTC 时间，在中国时区需要用+8小时就是本地时间，下面把 airflow 全面修改为中国时区。

    首先需要修改 airflow.cfg 文件，定位到 default_timezone 代码处，修改为 Asia/Shanghai

    [core]
...
default_timezone = Asia/Shanghai

    单单修改 ariflow.cfg 文件是不够的，需要修改源代码进行修正，不然即使界面显示的是国内时间，但其实内部定时任务还是会以 UTC 时间进行任务的调度。

    • 修改 airflow/utils/timezone.py 文件: 在 utc = pendulum.timezone('UTC') 这行(第27行)代码下添加如下代码，并修改 utcnow() 函数 (在第69行)，将 d = dt.datetime.utcnow() 修改为 d = dt.datetime.now()。
    from airflow import configuration as conf
try:
    tz = conf.get("core", "default_timezone")
    if tz == "system":
        utc = pendulum.local_timezone()
    else:
        utc = pendulum.timezone(tz)

    pass
    • 修改 airflow/utils/sqlalchemy.py 文件: 在 utc = pendulum.timezone('UTC') 这行(第37行)代码下添加:
    from airflow import configuration as conf

    9. Mac安装AirFlow的问题总结

    9.1 库之间不兼容

    Mac 下安装 airflow==1.10.3 版本时，出现安装的包与其他版本的包不兼容的情况，这里列出来记录一下。

    • dill : airflow==1.10.3 版本对 dill 版本要求是 0.2.2 <= ver.< 0.3，而安装最新的 pathos==0.2.4 要求 dill>=0.3.0，所以这里只能安装 pathos 的版本是 0.2.4。
    → pip install multiprocess==0.70.7
→ pip install pathos==0.2.4
    • markdown : airflow==1.10.3 版本对 markdown 版本要求是 2.5.2 <= ver.< 3.0，在你更新 jinja2 一些其他库时，可能会将 markdown 升级到了 3.0 以上的版本，所以建议恢复到 2.6.11。
    → pip install markdown==2.6.11

    9.2 安装 setproctitle 和 mysqlclient

    使用 pip 安装 setproctitle 和 mysqlclient 都会报错，具体报错信息大致如下:
