    GitHub搜索开源项目技巧

    GitHub 上有很多非常优秀的开源项目，下面分享一下搜索的技巧，学会这些技巧有助于快速找到预期的项目。

    1. in:name搜索

    可以使用 in:name <关键词> 搜索包含关键词项目名称，例如你可以使用以下命令搜索项目名称中包含 spring cloud 关键词的项目:

    in:name spring cloud

    2. in:description 搜索

    in:description <关键词> 用于搜索项目描述中包含关键词的项目，一般项目的描述都会比项目名称具备更多的信息，所以比起 in:name 会更好用一点。

    in:description spring colud

    3. in:readme搜索

    因为每个项目一般都会包含 README.md 文件，也可以通过这个来搜索你想要的项目:

    in:readme spring cloud

    4. 根据项目的 star 或者 fork 量搜索

    一般我们想要找一些比较火的开源项目，那么项目的 star 或者 fork 数量其实就是一个很好的参考。

    • 使用 stars:>数字 <关键词> 来搜索 star 大于等于 3000 的项目
    stars:>3000 spring cloud
    • 也可以使用 stars:数字1..数字2 <关键词> 来搜索 star 在固定区间的项目
    stars: 100..1000 spring cloud

    fork 的使用方法其实与 stars 方法类似，其实就将上述命令中 stars 换成 forks 就可以了。

    5. 根据项目的大小搜索

    可以使用 size:>数字 <关键词> 来搜索项目大小符合你期望的项目，有时候项目太大反而不能快速上手，从一些简单的 DEMO 着手一般会事半功倍。

    size:>=5000 spring cloud

    值得注意的是，这里数字代表的 K，即 5000 表示 5M。

    6. 根据项目更新频率搜索

    在找项目的时候，一定要注意该项目是否还在更新，或者说什么时间创建的该项目。

    • push:>日期 可以用来搜索更新时间，比方我们要搜索 2019-01-01 之后有过更新的项目:
    pushed:>2019-01-01 spring cloud
    • created:>日期 可以用来搜索创建时间，比方我们要搜索 2019-01-01 之后才创建的项目:
    created:>2019-01-01 spring cloud

    7. 按照编程语言进行搜索

    可以使用 language:<编程语言> 来按照语言来搜索项目，比如我们想要搜索以 python 作为开发语言的项目:

    language:python

    8. 根据作者来搜索

    那么是否可以更具作者来搜索他的项目呢，答案肯定是可以的。比如我们搜索莫烦的 GitHub 项目，一般可以直接找到他的 GitHub，然后一个一个查看。当然也可以直接通过 user:作者 指令进行搜索:

    user:MorvanZhou

    进一步可以结合一些其他的条件来进一步筛选，比方我们想搜索 NLP 的相关内容

    user:MorvanZhou language:python

    9. 根据组织来搜索

    可以使用 org:组织名 来搜索具体组织的仓库:

    org:spring-cloud

    10. 根据LICENSE来搜索

    这个不是非常有用，但是不是每一个开源项目都是免费的，必要的时候我们还是会用到根据 LICENSE 来搜索项目:

    license:apache-2.0 spring cloud
