    1.1 Cloudera & CDH 简介

    Cloudera 由来自 Facebook、谷歌和雅虎的前工程师杰夫.哈默巴切(Jeff Hammerbacher)、克里斯托弗.比赛格利亚(Christophe Bisciglia)、埃姆.阿瓦达拉(Amr Awadallah) 以及现任CEO甲骨文高管迈克.奥尔森(Mike Olson) 在2008年创建。

    1.1.1 公司发展历史

    • 2008年，Cloudera创立

    • 2009年

      • Hadoop 创始人 Doug Cutting 加入 Cloudera
      • Cloudera 发行 CDH 第一个商业化 Apache Hadoop 分发版

    • 2010年，Cloudera Manager 第一个 Hadoop 管理应用界面

    • 2011年

      • Cloudera 达到 100 运行客户
      • Cloudera University 扩展到 140 个国家

    • 2012年

      • Cloudera Enterprise 4 成为企业中的 Hadoop 标准
      • Cloudera Connect达到300多个商业合作伙伴

    • 2013年

      • Cloudera Impala、Cloudera Navigator、Cloudera Search
      • Tom Reilly成为首席执行官，Cloudera Connect包含 800 多个商业合作伙伴

    • 2014年，企业级数据大量采用

    1.1.2 Hadoop 版本

    Hadoop 版本主要有三个，分别是:

    • Apache(最原始的版本，所有发行版均基于这个版本进行改进)
    • Cloudera 版本(Cloudera’s Distribution Including Apache Hadoop，简称CDH)
    • Hortonworks 版本(Hortonworks Data Platform，简称 HDP)，对于国内而言，商业版本绝大多数选择 CDH 版本和原生态 Apache)

    1.1.3 Cloudera 5 综述

    • Cloudera 企业级数据中心蓝图
    • 更多样的工作负载(不仅仅是批处理)
      • 工作负载管理
      • 分析SQL(Impala)
      • 搜索
      • Spark
      • 第三方应用
      • 灵活的部署方式
    • 安全
    • 管治
    • 管理

    1.2 CDH的支持版本和系统要求

    支持的操作系统 : Hadoop(cdh) 支持以下操作系统，兼容RHEL

    • Red Hat Enterprise Linux 和 Centos
      • 5.7，64位
      • 6.4，64位
      • 6.4，使用 SE Linux 模式
      • 6.5，64位
      • 6.6，64位
    • SLES-- SUSE Linux Enterprise Server 11，64位 (SUSE Linux Enterprise Software Development Kit 11 SP1)
    • Debian–Wheezy 7.0和7.1，Squeeze6.0 (已停用)，64位
    • Ubuntu–Trusty14.04、Precise12.04、Lucid10.04 (已停止)，64位

    支持的JDK版本 : 支持 Oracle JDK 版本 1.8.0_11、1.7.0_67 和 1.6.0_31 并可选择在安装和升级过程中安装后两种版本

    支持的数据库 :

    • MySQL - 5.0、5.1、5.5 和 5.6
    • Oracle 11gR2
    • PostgreSQL - 8.4、9.1、9.2 和 9.3

    资源需求 :

    • 磁盘空间要求
      • /var 的分区上有 5GB 可用空间
      • /usr 的分区上有 500 MB 可用空间
    • RAM : 建议8 GB 以上内存，否则可能会因为 RAM 消耗过多而终止服务器
    • Python : 软件包安装需要Python 2.6 或 2.7

    1.3 系统环境准备

    • 设置 IP
    → vi /etc/sysconfig/network-scripts/ifcfg-eth0
IPV6INIT = "no" /ONBOOT = "yes"
    • 设置 hosts 文件
    → vi /etc/hosts
# 服务器名必须为完整的"域名"格式，如: cdh1.cdh.com
    • 关闭 selinux
    → getenforce
→ vi /etc/selinux/config
# 将SELINUX=enforcing 改为 SELINUX=disabled
    • 关闭防火墙，执行完后重启生效
    → chkconfig iptables off
    • 设置服务器名
    → vi /etc/sysconfig/network
# Networking=yes / hostname=cdh1.cdh.com
    • 配置互信
    → ssh-keygen -t rsa
→ ssh-copy-id -i id_rsa.pub 127.0.0.1
# 把 /root/.ssh/ 目录下的所有文件复制到集群中的其他服务器同目录下

    • 运行 CM 软件卸载包 uninstall-cloudera-manager.sh

    • 删除 cloudera 的安装目录（程序、环境变量…）

    • 卸载 cloudera 的 rpm 包

    • 停掉相关进程，清理 cloudera 相关文件

    3. CDH 的 Impala

    3.1 原理和架构

    查询执行流程 : Impalad 通过 odbc / jdbc 等接受到查询

    Coordinator 将查询计划分发到其他节点执行

    Coordinator 接收查询结果并发回给客户端

    3.2 CDH 的 Impala与Hive

    Hive :

    • 以MapReduce为执行引擎
    • 高延迟，低吞吐量
    • 通过Hadoop获得容错性
    • Java运行时使其能够动态加载文件格式解析函数以及UDFs

    Impala :

    • 传统的迭代器模型，流式数据处理
    • 容错性差
    • 流式执行引擎为高效实时查询奠定基础

    Impala 相对 Hive 的优势:

    • Impala 跳过 HDFS 直接读取磁盘数据，因此获得更高的磁盘带宽(对于 IO 密集型能够带来 3-4 倍的提升)
    • 对于需要多个 MapReduce 阶段的查询语句，Impala 能够获得更高的效率提升
    • 基于内存数据集的查询，Impala 能够获得更高的性能(100倍)

    Impala与Hive集成 : 批量与实时查询共存

    3.3 业务应用于运维

    3.3.1 运用Impala Shell

    Impala Shell : 支持交互式查询的文本的查询的文本行界面

    服务端Impala的启动(示例) : 在机器 mr1 上运行 statestored

    GLOG_v =1 nohup /usr/bin/statestored -state_store_port=24000

    如果 statestore 正常启动，可以在 /tmp/statestored.INFO 查看。如果出现异常，可以查看 /tmp/statestored.ERROR 定位错误信息

    3.3.2 查询语言

    Impala 的查询语言是基于 Hive 的 HiveQL，目前 Impala 不支持针对表和分区的 DDL，但是支持 DML

    大部分的 HiveQL 不需要修改就可以在 Impala 上面执行的，包括 JOIN、AGGREGATE、DISTINCT、UNION、 ALL、ORDER BY、LIMIT 和 subquery 等等

    3.3.3、Impala小结

    • 分布式环境下通用 SQL 引擎: 既支持 OLTP 也支持 OLAP
    • SQL 查询的规模和粒度: 从毫秒级到小时级
    • 底层存储依赖 HDFS 和 HBase
    • 使用更加高效的 C++ 编写
    • SQL 的执行引擎借鉴了分布式数据库 MPP 的思想而而不再依赖 MapReduce
