    18. 文件忽略

    18.1 全局忽略

    → touch  ~/.gitignore_global
→ git config --global core.excludesfile ~/.gitignore_global

    在 .gitignore_global 文件中，添加规则

    18.2 项目忽略

    1. 对于每一级工作目录，创建一个 .gitignore 文件
    2. 向该文件中添加要忽略的文件或目录
    3. 在创建并编辑这个文件之前，一定要保证要忽略的文件没有添加到 git 索引中
    4. .gitignore 文件同样可以像其它文件一样加到项目仓库里

    18.3 提交空文件夹

    一般情况下, 向空文件夹中添加 .gitkeep 文件, 并将其添加到版本控制, 即所在的文件夹也会被跟踪, 对文件这个文件名称, 可以任意指定, 一般习惯指定这个名字。

    18.4 忽略规范

    1. 所有空行或者以注释符号 # 开头的行都会被 Git 忽略
    2. 匹配模式最后跟反斜杠(/)说明要忽略的是目录
    3. 要忽略指定模式以外的文件或目录，可以在模式前加上惊叹号(!)取反
    4. 星号(*)匹配零个或多个任意字符
    5. 问号(?)只匹配一个任意字符
    6. [abc] 匹配任何一个列在方括号中的字符
    7. 如果在方括号中使用短划线分隔两个字符，表示所有在这两个字符范围内的都可以匹配(比如 [0-9])

    18.5 忽略.log日志文件

    → touch test.log && echo "test.log" >> .gitignore
→ git st
On branch dev
Untracked files:
  (use "git add <file>..." to include in what will be committed)

    .gitignore

nothing added to commit but untracked files present (use "git add" to track)

    可以看到，这里只提示 .gitignore 文件未被跟踪，而 test.log 没有提示。

    建议大家，将 .gitignore 提交到版本中，因为每个项目忽略的文件不一样。

    当然也可以不提交，你可以把自己添加进来。在 .gitignore 中添加:

    test.log
.gitignore

    18.6 .gitignore文件

    我们将 .gitignore 中内容全部删除掉。

    → git add test.log
→ git ci -am "an test.log"

    此时 test.log 已经添加到了版本控制中，一旦添加到版本控制中，忽略文件就不起作用了，即使你后来添加了test.log，也不会起作用。

    → git rm -cached test.log

→ cd lib/

→ git st

nothing to commit, working directory clean

    可以看到，这里 lib 文件夹下的 test.log 也被忽略了。
