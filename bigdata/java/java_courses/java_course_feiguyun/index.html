    1.1.1 Java语言简介

    Java是 Sun(Standford University Network) 公司推出的一种编程语言。

    1.4.3 类的继承

    • 类与类之间还有一种父与子的关系，子继承父类(超类)方法与属性，这个我们就称之为继承，继承体现了类之间的泛化关系
    • 子类拥有父类的一些公有的属性和方法，并且子类可能也会有自己的方法和编程
    • 除了添加父类中没有的新方法外，子类还可以通过重写父类的方法重定义自己的行为方式

    1) 继承的语法

    [类的修饰符] class <子类名> extends <父类名>{
     <属性定义>;
     <方法定义>;
}
    • 在定义父类时用 extends 关键字指明新定义的父类，就在两个类之间建立了继承关系。新定义的类称为子类，它可以从父类那里自动继承所有非 private 的属性和方法作为自己的成员
    • 子类永远不会继承父类的构造方法。因为构造方法永远和类名相同
    • 继承了什么？除了构造方法之外，父类的所有非 private 方法和非跨包的默认属性都由子类的对象继承

    1.4.6 Java异常处理机制

    异常是指程序在运行时发生的错误。通过异常处理机制，我们可以用一种结构化的可控方式来处理运行时错误。

    尽管大多数高级程度设计语言都提供了一些异常处理机制，但是Java中的异常处理机制比其他语言提供的更简洁、更灵活。

    1) 错误和异常

    在 Java 中，所有的异常都是用类来表示。所有的异常类都是从 java.lang 包中的Throwable 类继承而来的。用户也可以自己定义异常类型。

    • Error

    Error 用来表示编译和运行错误，如程序进入了死循环、内存溢出等。

    Error 只能在编译阶段解读，运行时程序本身无法解决，只能依靠其他程序的干预，否则会一直处于非正常状态。

    Error 处理一般由系统承担，Java 本身不提供相应的 Error 处理机制。

    • Exception

    Exception 用来表示由程序活动所导致的错误，如运算时除数为 0、打开一个文件时发现文件不存在等。

    当程序运行出现 Exception 时，在程序中加入异常处理代码，仍可使程序继续运行直至程序结束。异常处理机制正是针对 Exception 类的。

    2) 异常处理格式

    程序发生异常时，会生成一个异常对象，这个异常对象会在方法内部被抛出，它假设异常将被异常处理程序屏蔽。

    如果程序不捕获异常，它就要被 JVM 捕获，这个方法将在抛出异常的过程中结束。

    要是不希望就此结束，可以在方法内部设置一个特殊的块来监视可能产生异常的代码，然后让程序来捕获并处理异常对象从而不会影响其他语句的执行。

    在 Java 中，提供了 try 和 catch语句来捕获和处理一个或多个异常。语法格式如下:

    try {
	// 可能产生异常的代码
}
catch(Type1 e) {
	// 对异常类型 Type1 的异常处理代码
}

    对于一些代码，可能会希望无论try块中的异常是否被抛出，他们都可能得到执行。

    为了在退出 try / catch 代码块时指定一个要执行的代码块，可以在异常处理程序后面加上 finally 子句。

String[] strName = {"I", "like", "Java"};

    2.2.2 ArrayList 和 LinkedList

    ArrayList 其实是包装了一个数组 Object[]，当实例化一个 ArrayList 时，一个数组也被实例化，当向 ArrayList 中添加对象时，数组的大小也相应的改变。

    ArrayList 可提供快速随机访问。你可以随机访问每个元素而不用考虑性能问题，通过调用 get(i) 方法来访问下标 i 的数组元素。ArrayList 随机插入对象的速度慢，当你要向数组中任意两个元素中间添加对象，数组需要移动所有后面的对象。

    LinkedList 是通过节点直接彼此连接来实现的。每一个节点都包含前一个节点的引用，后一个节点的引用和节点存储的值。当一个新节点插入时，只需要修改其中其中保持先后关系的节点的引用即可，当删除记录时也一样。

    LinkedList 操作其中对象的速度快。只需要改变连接，新的节点可以在内存中的任何地方。LinkedList 不能随机访问，虽然存在 get() 方法，但是这个方法是通过遍历节点来定位的，所以速度慢。### 2.2.3 Java中的泛型

    2.4.1 JDBC 的作用

    JDBC(Java Database Connectivity) 是 Java 数据库连接API。实现对数据库资源的访问。

    简单地说，JDBC能完成三件事，主要的类别为 java.sql.*;

    • 与一个数据库建立连接
    • 向数据库发送 SQL 语句
    • 处理数据库返回的结果

    2.4.2 JDBC 的基本概念

    • DriverManager: 负责选取数据数据库驱动程序和建立新的数据库连接

    • Driver: 将应用程序的 API 请求转换为特定的数据库请求

    • Connetcion: 将应用程序连接到特定的数据库

    • Statement(PreparedStatement): 在一个给定的连接中，用于执行一个静态(动态) 的数据库SQL 语句

    • ResultSet(可以没有，如在执行更新操作后): SQL 语句执行完后，返回的数据结果集(包括行、列)

    4.2 HDFS 与 MapReduce

    Hadoop的框架最核心的设计就是: HDFS 和 MapReduce。HDFS 为海量的数据提供了存储，而MapReduce为海量的数据提供了计算框架。

    4.2.1 Hadoop Distributed File System(HDFS)

    对外部客户访问而言，HDFS 就像一个传统的分级文件系统。可以创建、删除、移动或重命名文件等等。但是 HDFS 的架构是基于一组特定的节点构建，这些节点包括一个 NameNode 和多个 DataNode。其中 NameNode 在HDFS内部提供元数据服务，DataNode 为 HDFS 提供存储块。

    存储在 HDFS 中文件首先被分成块，然后将这些块复制到多个计算机中(DataNode)，块的大小(通常为 64MB) 和复制的块数量在创建文件时由客户机决定。NameNode 可以控制所有文件操作。HDFS 内部的所有通信都基于标准的 TCP/IP 协议。

    4.2.2 MapReduce 计算框架

    MapReduce 是大规模数据(TB) 计算的利器，Map 和 Reduce 是它的主要思想，来源于函数式编程语言。其中，Map 负责将数据打散，Reduce 负责对数据进行聚集，用户只需要实现 map 和 reduce 两个接口，即可完成 TB 级数据的计算，常见的应用包括: 日志分析和数据挖掘等数据分析应用。另外，还可用于科学数据计算，如圆周率 PI 的计算等。

    Hadoop MapReduce 的实现也采用了 Master/Slave 结构。Master 上的进程名叫做 JobTracker，而 Slave 上的叫做 TaskTracker。用户提交的计算叫做 Job，每一个 Job 会被划分为若干个 Tasks。JobTracker 负责 Job 和 Tasks 的调度，而 TaskTracker 负责执行 Tasks。

    4.2.3 MapReduce 计算框架的特点

    • 移动计算而非移动数据，化整为零，分片处理
    • 本地化计算，并行IO，降低网络通信

    4.4.1 MapReduce 处理原理

    MapReduce 计算模式是一种标准的函数式编程模式。其核心在于可以将一个函数作为参数传递给另一个函数，这样一种函数通常被称作高次函数。通过多个高次函数的串接，可以将数据的计算过程转化为一系列函数的执行过程

    MapReduce 计算模式将数据计算过程分为两个阶段: Map 和 Reduce。这两个阶段定义了两个处理函数，分别是 mapper 和 reducer。

    在 Map 阶段，原始数据被输入 mapper 进行过滤和转换，获得的中间数据在 Reduce 阶段作为 reducer 的输入，经过 reducer 的集合处理，获得最终的处理结果。

    为了适应多样化的数据环境，MapReduce 中把键/值数据对(Key-Value pair) 作为基础数据单元。Key 和 Value 可以是简单数据类型，比如整数、浮点数、字符串等；也可以是复杂数据结构，比如列表、数组、自定义结构等。

    • Map: <k1, v1> -> {<k2, v2>}

    • Reduce: <k2, {v2}> -> {<k3, v3>}

    其中，<...> 代表键/值数据对，{...} 代表 List。

    public static class WordCountMapper<K extends WritableCompare, V extends Wrieable>
	extends MapReduceBase implements Mapper<Object, Text, Text, IntWritable> {
    private final static IntWritable one = new IntWritable(1);
    private Text word = new Text();
    
    @override
    public void map(
    	Object key,
        Text value,
        OutputCollector<Text, IntWritable> output,
        Reporter reporter)
        throws IOException {
        String[] words = value.toString.split(" ");
        
        for (String str: words) {
            word.set(str);
            output.collect(word, one);
        }
    }
}

    @override
    public void reduce(
    	Text key,
        Iterator<IntWritable> values,
        OutputCollector<Text, IntWritable> output,
        Reporter reporter)

        String[] words = value.toString.split(" ");
        
        int total = 0;
        while (values.hasNext()) {
            values.Next();
            total++;

        output.collect(key, new IntWritable(total));

    5.2.4 HDFS数据块的概念

    • 在操作系统中，每个磁盘都有默认的数据块大小，这是磁盘进行数据读/写的最小单位。文件系统块一般为几 KB，而磁盘块一般为 512 字节。这些信息对于用户来说是透明的

    • HDFS 同样也有块(block) 的概念，但是大得多，默认是 64MB。一个以 HDFS 格式保存的文件，会被分成 64M 的整数个块来存储，但是存储 HDFS 中小于一个块大小的文件不会占据整个块的空间。通过配置 hdfs-core.xml 文件中的属性 ds.block.size 的值可以调整块的大小

    • HDFS 的块比磁盘的块大，其目的是为了最小化寻址开销。MapReduce 中的 map 任务通常一次只处理一个块中的数据，与磁盘文件系统相似，HDFS 中 fsck 指令可以显示块信息

    → hadoop fsck /test/php.chm -files -blocks

    8.1 HBase简介及核心概念

    8.1.1 HBase 和 Hadoop之间的关系

    HBase(Hadoop DataBase) 是一个高可靠、高性能、面向列、乐伸缩的分布式数据库系统。它使用类似 GFS 的 HDFS 作为底层文件存储系统，在其上运行 MapReduce 批量处理数据，使用 ZooKeeper 作为协同服务组件。

    HBase 项目使用 Java 语言实现，最初是由 Google BigTable 原型演化而来，2007 年第一个简单可用的 HBase 版本发布，2008 年 1 月，Hadoop 升级为 Apache 的顶级项目时，HBase 作为Hadoop 的子项目存在。后来随着 Hadoop 版本的提升而不断更新，2010 年 5 月 HBase 成为 Apache 顶级项目，截止 16 年底 HBase 版本是 0.96 和 1.2。HBase 的运行严重依赖于Hadoop，且二者版本存在协调关系。

    8.1.3 HBase模型

    1) 表和列族

    HBase 是一种列式存储的分布式数据库，其核心概念是表(Table)。和传统的数据库一样，表由行和列组成，但 HBase 把不同的列组成一个列族(Column Family)，同一列的值又可以有多个版本(version)，这种组织形式主要是为了存取性能的考虑。

    表: 在 HBase 中数据以表的形式存储。表名使用 Java String 类型类型或 byte[] 字节数组表示。每个表名对应HDFS中的一个目录结构。创建表时需要指定表名和至少一个列族。列族影响表的物理存储结构。

    列族: 是一些列的集合。创建好不能频繁修改，建议一个表只设定一直两个。列族名由可见字符组成。列族中包含列的数量没有限制，可以有数百万个列。列值没有类型和长度的限制。一个列族的所有列存储在同一个底层文件(HFile) 中。常见的引用列的格式是 family:qualifier，qualifier 是任意的字节数组。

    2) 行键

    **行键(**Rowkey) 是 HBase 中最重要的概念之一，它在表中用来唯一的标识一行，其值保存为二进制字节数组。表中的行是按照行健的字典序排序存储的。

    在 HBase 中行键是唯一的索引，所以在设计键行键时必须考虑检索数据的效率，比如把常用的检索字段组合作为 rowkey，并且要保证其唯一性，但 rowkey 长度又不能太长。太长的 rowkey 会增加存储开销，降低内存利用率，从而降低索引命中率。另外 rowkey 的设计同时要考虑如何其值能够尽量散列，这样会保证所有的数据不都是在一个 region 上，避免做读写时负载集中在个别 region 上面。

    3) 单元和时间版本

    单元(Cell) 是行键、列族和列名一起确定了一个单元。存储在单元里的数据称为单元值(Value)。单元值没有数据类型，存储为字节数组 byte[]。

    时间版本(version)，单元值有时间版本，时间版本用时间戳标识，是一个 long 型数字，默认使用当前时间戳。HBase 保留单元值时间版本的数量基于列族进行配置，默认数量是 3 个。下面展示如何保存两个不同版本的数据:

    使用 Table 对象结合 Get 对象可以实现按照指定行键得到对应列族和列值。

    public static void getByRowkey(String rowkey) throws IOException {
    HBaseUtil.init();
    String tableName = "room_log2";
    Table table = HBaseUtil.connection.getTable(TableName.valueOf(tableName));
    Get get = new Get(Bytes.toBytes(rowkey));
    Result result = table.get(get);
    Cell[] cells = result.rawCells();
    
    for (Cell cell: cells) {
        System.out.println("行键=>" + new String(CellUtil.cloneRow(cell)));
        System.out.println("列名=>" + new String(CellUtil.cloneFamily(cell)) + ":"
			+ String(CellUtil.cloneQualifier(cell)));
        System.out.println("列值=>" + new String(CellUtil.cloneValue(cell)));
        System.out.println("版本号=>" + cell.getTimestamp());
    }
}

    public static void deleteByRowkey(String rowkey) throws IOException {
    HBaseUtil.init();
    String table = "room_log2";

    HBaseUtil.close();
}

    Table.delete() 方法参数还支持 List<Delete> 列表格式，即同时可以执行多个删除动作，类似于 put() 方法。HTable 表提供了一个 CAS 的原子性操作 checkAndDelete() 函数，该函数在多个客户端对一行数据进行操作时进行修改。

    HBase 表中更新操作是针对某个列的，不是针对行的。由于在创建的时候可以为列族中的列值指定多个保存版本的(VERSIONS)。如果该属性未设置，则按照行键和列名更新列值时，会覆盖原来的值，因为 HBase 表没有保存多个版本的值。

    如果创建表时指定了 VERSION 值，更新列值时并不会覆盖原有值，而是新插入一个值，只是这个值的时间戳和原有的不一致。在 Get 操作时默认取得最新时间戳所对应的列值。以下演示如何在HBase客户端更新和获取多版本的列值。

    如果想要更新某个 cell 的值，只需要使用 Put 方法往 Table 对象中插入数据即可。以下代码按照指定 rowkey 更新了 cf:address 字段的值。

    // 更新列值
public static void updateByRowkey() throws IOException {
    HBaseUtil.init();
    String tableName = "room_log2";
    String rowkey = "2011-2-15 10:01:57_孙某某_12345671111";
    String cf = "cf";
    String cq = "address";
    String newval = "浙江句容";
    Table table = HBaseUtil.connection.getTable(TableName.valueOf(tableName));
    // 按照rowkey更新某列的值
    Put put = new Put(Bytes.toBytes(rowkey));
    put.addColumn(Bytes.toBytes(cf), Bytes.toBytes(cq), Bytes.toBytes(newval));
    table.put(put);
    table.close();
    HBaseUtil.close();
    System.out.println("update OK.");
    getByRowkey(rowkey);
}

    使用 Get 或 Scan 方法得到 cell 的值，始终默认是 timestamp 最新的那个。如果想得到历史 cell 的值，可以使用 Get.setMaxVersions() 得到所有版本的 cell 值。

    public static void getAllVersion(String rowkey) throws IOException {
    HBaseUtil.init();
    Table table = HBaseUtil.connection.getTable(TableName.valueOf(tableName));
	// 构造一个查询对象，按照行键进行检索
    Get get = new Get(Bytes.toBytes(rowkey));
    get.setMaxVersion();  // 得到所有版本的值
    
    Result result = table.get(get);
    Cell[] cells = result.rawCells();
    
    for (Cell cell: cells) {
        System.out.println("行键=>" + new String(CellUtil.cloneRow(cell)));
        System.out.println("列名=>" + new String(CellUtil.cloneFamily(cell)) + ":"
			+ String(CellUtil.cloneQualifier(cell)));
        System.out.println("列值=>" + new String(CellUtil.cloneValue(cell)));
        System.out.println("版本号=>" + cell.getTimestamp());
    }
    table.close();
    HBaseUtil.close();
}

    8.4 使用Filter过滤器筛选数据

    使用 get 或 scan 方式可以实现数据筛选，但是粒度太粗。如果想实现对行键的筛选，可以配合 Filter 过滤器完成，HBase 提供了针对不同维度使用的过滤器类。

    过滤器名 使用场合
    行过滤器(RowFilter) 行过滤器基于行键来过滤数据，配合比较运算符来返回符合条件的行键
    列族过滤器(FamilyFilter) 通过比较列族的名称来筛选结果
    列名过滤器(QualifierFilter) 通过比较列的名称(不是比较列值) 来筛选结果
    列值过滤器(ValueFilter) 筛选包含某个特定的所有单元格，并返回列数据
    单列值过滤器(SingleColumnValueFilter) 筛选包含某个特定值的特定单元格，并返回整行数据
    分页过滤器(PageFilter) 通过指定 pageSize 参数实现对检查结果按行分页
    行键过滤器(KeyOnlyFilter) 在行键中筛选是否包含某个特定值，并只返回行键值
    时间戳过滤器(TimestampFilter) 实现对单元格版本进行筛选
    随机行过滤器(RandomRowFilter) 可以实现随机返回结果集中的行

    HBase 提供了功能丰富的过滤器功能丰富的过滤器类，在实际应用中可以在一个检索中同时设定多个过滤器，实现多条件组合查询效果。通过使用 FilterList 实现该功能，并且可以设定多个 Filter 之间是 AND 或 OP 的关系。

    过滤器中有一种叫 CompareFilter 过滤器，它提供的比较运算符可以指定比较对象之间的类型，如下表所示:

    操作 描述
    LESS 匹配小于设定值的值
    LESS_OR_EQUAL 匹配小于或等于设定值的值
    EQUAL 匹配等于设定值的值
    NOT_EQUAL 匹配不等于设定值的值
    GREATER_OR_EQUAL 匹配大于或等于设定值的值
    GREATER 匹配小于设定值的值
    NO_OP 排出一切值

    使用 RowFilter 实现对行键值的 BETWEEN...AND 和 LIKE 检索

    String tableName = "room_log2";
Table table = HBaseUtil.connection.getTable(TableName.valueOf(tableName));
// 构造一个查询对象，按照行键进行检索
FilterList filterList = new FilterList(FilterList.operator.MUST_PASS_ALL);
// 按照开始时间段对入住时间进行 BETWEEN...AND 检索
String sd = "20110201 00:00:01_";
String ed = "20110831 23:59:59_";
filterList.addFilter(new RowFilter(
	CompareFilter.CompareOp.GREATER_OR_EQUAL,
	new BinaryPrefixComparator(Bytes.toBytes(sd))));
filterList.addFilter(new RowFilter(
    CompareFilter.CompareOp.LESS_OR_EQUAL,
	new BinaryPrefixComparator(Bytes.toBytes(ed))));
// 对姓名进行模糊查找
filterList.addFilter(new RowFilter(
    CompareFilter.CompareOp.EQUAL,
	new SubstringComparator("孙")));
Scan scan = new Scan();
scan.setFilter(filterList);
ResultScanner = resultScanner = table.getScanner(scan);

for (Result result: resultScanner) {
    for (Cell cell: result.listCells()) {
        System.out.println("行键=>" + new String(CellUtil.clonerRow(cell)));
    }
}

    使用 SingleColumnValueFilter 实现对 cell 值的模糊检索

    HBaseUtil.init();
String tableName = "room_log2";
Table table = HBaseUtil.connection.getTable(TableName.valueOf(tableName));
// 构造一个查询对象，对列值进行检索
String cf = "cf";
String cq = "address";
String keyword = "市";

SingleColumnValueFilter filter = new SingleColumnValueFilter(
	Bytes.toBytes(cf),
    Bytes.toBytes(cq),
    CompareFilter.CompareOp.EQUAL,
    new SubstringComparator(keyword);
);
filter.setFilterIfMissing(true);

Scan scan = new Scan();
scan.setFilter(filter);
ResultScanner resultScanner = table.getScanner(scan);

for (Result result: resultScanner) {
    for (Cell cell: result.listCells()) {
        System.out.println("行键=>" + new String(CellUtil.cloneRow(cell)));
    }
}
