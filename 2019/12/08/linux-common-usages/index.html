    1. nohup后台挂起任务

    1.1 终端运行

    可以使用 nohup 来将 jupyter 挂起在后台:

    → nohup jupyter notebook --ip='*' --no-browser > jupyter.log 2>&1 &

    1.2 脚本后台挂起

    上面终端运行的方式，其实还是需要按下回车键才能将任务在后台挂起。下面介绍一种脚本方式，可以避免这个问题:

    #!/bin/bash

str=$"\n"
nohup jupyter notebook --ip='*' --no-browser > /dev/null 2>&1 &
sstr=$(echo -e $str)
echo $sstr

    2. 安装ohmyzsh

    2.1 安装 zsh

    → yum install -y zsh

    安装完成后，将默认的 bash 切换到 zsh。

    → chsh -s /bin/zsh

    2.2 安装 oh ~my ~zsh

    假设大家已经安装了 git 工具了，这里在终端中指向以下代码:

    → sh -c "$(curl -fsSL https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"

    至此，oh~ my~ zsh 安装完成。

    3. Centos源码方式安装 gcc

    → wget wget http://ftp.gnu.org/gnu/gcc/gcc-7.3.0/gcc-7.3.0.tar.gz
→ tar -zxvf gcc-7.3.0.tar.gz
→ cd gcc-7.3.0
→ ./contrib/download_prerequisites
→ cd ..
→ mkdir gcc-build-7.3.0
→ cd gcc-build-7.3.0
→ ../gcc-7.3.0/configure --enable-checking=release --enable-languages=c,c++ --disable-multilib
→ make -j4
→ make install

    但是使用同样方法安装 gcc-8.3.0 会报错。检查是否安装成功: 

    [root@promote ~]# g++ -v

    4. Centos更新系统时间

    首先修改时区，找到相应的时区文件 /usr/share/zoneinfo/Asia/Shanghai 替换当前的 /etc/localtime。 

    → yum -y install ntp ntpdate

    在使用 ntpdate 进行时间更新的时候报错，出现 Can’t adjust the time of day: Operation not permitted，应该是在一些虚拟 vps上，可能存在一些限制，导致此处报错。

    解决办法:

    → cp /usr/share/zoneinfo/Asia/Shanghai /etc/localtime

    修改 /etc/sysconfig/clock 文件的内容为： 

    ZONE=”Asia/Shanghai” 
UTC=false 
ARC=false
